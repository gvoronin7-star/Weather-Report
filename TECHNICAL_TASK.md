# Weather Report - Техническое задание

## 1. Описание приложения

**Название:** Weather Report  
**Тип:** Консольное приложение с GUI интерфейсом  
**Назначение:** Отображение текущей погоды по названию города или координатам

## 2. Функциональные требования

### 2.1 API интеграция
- **Geocoding API:** преобразование названия города в координаты (бесплатный)
  - Endpoint: `https://api.openweathermap.org/geo/1.0/direct`
  - Параметры: `q`, `limit=1`, `lang=ru`
  - Возвращает: `lat`, `lon`, `name`, `country`, `state`

- **Current Weather API:** получение текущей погоды по координатам (бесплатный)
  - Endpoint: `https://api.openweathermap.org/data/2.5/weather`
  - Параметры: `lat`, `lon`, `units=metric`, `lang=ru`
  - Возвращает: `temp`, `feels_like`, `humidity`, `description`, `icon`

### 2.2 Режимы ввода
1. **По городу** - пользователь вводит название города
2. **По координатам** - пользователь вводит широту и долготу
0. **Выход** - завершение программы

### 2.3 Обработка множественных результатов
Если найдено несколько городов с одинаковым названием - предложить выбор с указанием страны и региона

### 2.4 Кэширование
- Сохранение последнего успешного ответа в `weather_cache.json`
- Метаданные: `city`, `lat`, `lon`, `fetched_at` (ISO timestamp)
- При сетевой ошибке - предложить использовать кэш, если данные младше 3 часов

### 2.5 Обработка ошибок и ретраи
- Статус ответа ≠ 200 - понятное сообщение без трейсбека
- Пустой ответ геокодинга - "Город не найден"
- Невалидный API ключ - "Ошибка авторизации. Проверьте API_KEY в .env"
- HTTP 429 (rate limit) или временные ошибки - до 3 повторов с экспоненциальной паузой (1s, 2s, 4s)

### 2.6 CLI интерфейс
Вывод: `Погода в <город>: <температура>°C, <описание>`

### 2.7 GUI интерфейс
- Окно с полем ввода города
- Кнопки: "По городу", "По координатам"
- Отображение результата с иконкой погоды
- Кнопка для запуска GUI из CLI


### Модуль weather_app.py
- `load_api_key()` - загрузка ключа из .env
- `get_coordinates(city: str) -> tuple[float, float]` - геокодинг
- `get_weather_by_coordinates(lat: float, lon: float) -> dict` - погода по координатам
- `get_weather_by_city(city: str) -> dict` - погода по городу (с выбором при множественных результатах)
- `save_cache(data: dict)` - сохранение в кэш
- `load_cache()` - загрузка из кэша
- `make_request_with_retry(url: str, params: dict) -> dict` - запрос с ретраями
- `run_cli()` - CLI интерфейс
- `run_gui()` - GUI интерфейс
